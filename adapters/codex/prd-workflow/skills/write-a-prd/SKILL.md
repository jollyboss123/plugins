# Write a PRD

## Outcome
Produce a PRD with these sections:
- Problem Statement
- Solution
- User Stories
- Implementation Decisions
- Out of Scope
- Further Notes

## Workflow
1. Gather product intent, constraints, and success criteria.
2. Explore repository context to validate assumptions.
3. Resolve key ambiguities with focused clarification questions.
4. Draft the PRD in a handoff-ready format.

## Guardrails
- Prefer durable decisions over fragile implementation details.
- Avoid unnecessary file-level prescriptions unless explicitly requested.
