# PRD to Plan

1. Confirm the PRD source.
2. Explore architecture and integration layers.
3. Document durable decisions.
4. Break work into thin, verifiable vertical slices.
5. Produce a markdown plan with acceptance criteria per phase.

Prefer behavior-level phase descriptions over file-level commitments.
