# Write a PRD

1. Collect problem context, constraints, and desired outcomes.
2. Explore repository context to verify current behavior and constraints.
3. Clarify unresolved high-impact ambiguities.
4. Produce a PRD with: Problem Statement, Solution, User Stories, Implementation Decisions, Out of Scope, Further Notes.

Do not overfit to fragile implementation details unless explicitly requested.
